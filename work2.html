COGS Cafeteria and Catering Management System

A web application to support campus cafeteria to management the order and inventory

In this project, I have learned the 3 tier concept.
1. Presentation tier: User Interface developed with HTML, CSS, javascript, bootstrap, can run on web browser.
2. Application tier: infomation collected in this tier, developed with Python with Flask.
3. Data tier:the information processed by the application was stored in this tier, developed with MySQL.

Function of the application

Menu page: Use can select to manage the inventory or add new order.
1. Manage product
2. New Order


Delete inventory: User can delete the inventory in the manage product
Press OK after read the warning notice


Create inventory: User can add new product in the manage product page
Enter the product details then press Save button


New Order: user can back to menu and select New Order
To create a new order, user need to input their name at the customer name box

User now can press Add More to add new item in their order

After add finish added the items, press Save button to save the order
Now, the new order will storage into the database

The new order record can find in the MySQL database

Also, the new order will be display on the meun page after save
